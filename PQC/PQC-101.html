1. The Quantum Threat Model

Post-quantum cryptography means classical cryptographic algorithms designed to remain secure against both classical and quantum attackers. It does not require a quantum computer to run; it replaces vulnerable public-key math in today's software and hardware.

What breaks

RSA, finite-field Diffie-Hellman, elliptic-curve Diffie-Hellman, ECDSA, and EdDSA rely on math that Shor's algorithm threatens on a sufficiently capable quantum computer.

What weakens

Symmetric encryption and hashes are affected differently. Grover's algorithm changes security margins, so larger symmetric key sizes and hash outputs remain important.

Why now

Encrypted data with long confidentiality life can be captured today and decrypted later if it still matters when powerful quantum computers arrive.

Practical rule: prioritize data whose confidentiality lifetime is longer than your migration timeline. Medical, legal, government, telecom, finance, identity, firmware signing, and long-lived IP usually move first.

2. Current Standards and Algorithm Families

NIST finalized the first three Federal Information Processing Standards in 2024. These are the main algorithms to learn and plan around now. NIST also continues standardization work on backup and alternative algorithms.

Algorithm families

Lattice-based

Fast and practical for many uses. ML-KEM and ML-DSA are the main examples.

Hash-based

Conservative signature foundation. SLH-DSA is stateless and useful as a backup path.

Code-based

Different math from lattices. HQC gives defense-in-depth as a future backup KEM.

Other research families

Multivariate, MPC-in-the-head, and other candidates are still studied, especially for specialized signature needs.

3. Where PQC Shows Up

PQC is not a single library swap. It changes wire protocols, certificates, handshakes, signatures, key management, procurement, testing, and operations.

TLS, VPN, SSH

Key establishment is usually the first visible protocol impact. Hybrid modes reduce transition risk while ecosystems mature.

PKI and certificates

Signature sizes can affect certificate chains, CT logs, firmware constraints, and legacy parsers.

Code and firmware signing

Long-lived verification matters. Devices may need algorithm agility before PQC-only signatures become default.

Data at rest

PQC mainly changes public-key wrapping, exchange, and signatures. Symmetric encryption still matters and should use adequate key sizes.

4. Migration Roadmap

  1. Discover: create a cryptographic inventory of algorithms, libraries, certificates, protocols, key stores, and systems.
  2. Classify: map each use to confidentiality lifetime, exposure, business criticality, and regulatory pressure.
  3. Prioritize: handle long-lived secrets, external-facing protocols, identity, signing roots, and procurement gates first.
  4. Design agility: make algorithms configurable, support multiple key/signature types, and avoid hard-coded assumptions.
  5. Test hybrid: evaluate size, latency, middlebox behavior, rollback, monitoring, and interoperability.
  6. Deploy in waves: start with test environments and low-risk endpoints, then expand based on evidence.
  7. Monitor: track standards errata, library updates, validation status, and cryptanalytic developments.
Good migration evidence: inventory coverage, owner, crypto use, data lifetime, impacted protocol, replacement plan, test result, rollback plan, and target date.

6. Glossary

KEM

Key Encapsulation Mechanism. Lets two parties establish a shared secret over a public channel.

Hybrid mode

Combines classical and PQC mechanisms during transition so both must fail for the secret to fail.

Crypto agility

Ability to change algorithms, keys, parameters, and providers without redesigning the system.

CBOM

Cryptographic Bill of Materials: inventory of algorithms, certificates, libraries, keys, and protocol use.

Q-day

Informal name for the point when a quantum computer could break widely used public-key cryptography.

Harvest now, decrypt later

Attack pattern where encrypted traffic is collected now for possible future decryption.
