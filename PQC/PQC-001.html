Start Here: Security and Cryptography Basics

A non-assumed starting point for learners who need to understand what cryptography protects before learning why post-quantum cryptography matters.

  • Explain confidentiality, integrity, authentication, and signatures in plain English.
  • Distinguish encryption, hashing, MACs, digital signatures, and certificates.

1

What problem cryptography solves

Cryptography exists because modern systems communicate over networks they do not fully control. A phone, browser, router, server, cloud workload, SIM, or IoT device may be sending data through equipment owned by many different parties. Attackers may be able to watch traffic, copy old messages, inject new messages, impersonate a service, or modify files in transit.

The beginner mistake is thinking cryptography only means hiding messages. Hiding is one job, but trust is the broader purpose. Cryptography helps answer: who am I talking to, did this data change, is this update really from the vendor, did this transaction come from the private key holder, and can two systems agree on a fresh secret in public?

Post-quantum cryptography is not a new app category. It is a replacement and transition path for specific cryptographic jobs that depend on vulnerable public-key mathematics.

Attacker can observeNetwork traffic may be copied.
Attacker can modifyPackets and files can be changed.
Crypto verifiesSystems check identity and integrity.
Crypto protectsData is encrypted after trust is established.

Concrete example

When a browser opens a banking site, it checks a certificate, negotiates keys, encrypts the session, and verifies protected records. The user sees a login page, but the browser and server completed several trust-building steps first.

Common pitfall

Do not equate security with encryption alone. An encrypted connection to an impersonator is still unsafe.

Takeaway

Cryptography is a trust system, not just a secrecy tool.

2

The four common security jobs

Confidentiality means unauthorized parties cannot read the content. Integrity means changes can be detected. Authentication means a party can prove identity or possession of a secret. Non-repudiation means a signature can later prove a particular private key signed something.

A single product may use all four jobs at once. A secure software update can be public, so confidentiality may not matter, but integrity and signature verification are critical. A medical record stored in the cloud needs confidentiality and integrity. A cellular network needs authentication and key establishment.

PQC affects these jobs unevenly. Bulk encryption and hash functions remain important, while public-key key establishment and signatures need the most direct migration attention.

ConfidentialityKeep content private.
IntegrityDetect changes.
AuthenticationProve who or what.
SignatureProve origin over time.

Concrete example

A firmware update file may be downloadable by anyone. The device still refuses to install it unless the signature proves it came from the vendor and the bytes were not altered.

Common pitfall

Do not use the same word 'encryption' for every cryptographic job. That hides what actually needs to be migrated.

Takeaway

Name the cryptographic job before choosing an algorithm.

3

Symmetric versus public-key cryptography

Symmetric cryptography uses a shared secret. It is fast and practical for protecting large amounts of data once both sides already have the secret. AES-GCM is a common example for authenticated encryption.

Public-key cryptography uses a public key and a private key. It helps systems identify each other, establish shared secrets over public networks, and verify signatures. RSA, Diffie-Hellman, elliptic-curve Diffie-Hellman, ECDSA, and EdDSA are widely deployed classical public-key systems.

A sufficiently capable quantum computer running Shor's algorithm threatens the public-key systems above. Grover's algorithm affects symmetric security margins differently, so larger symmetric keys and hashes remain useful.

SymmetricFast after secret exists.
Public keySolves internet-scale trust.
Quantum issueThreatens classical public-key math.
MigrationReplace or hybridize public-key parts.

Concrete example

TLS uses public-key techniques to authenticate and establish secrets, then uses fast symmetric encryption for the actual data stream.

Common pitfall

Do not say PQC replaces AES. PQC mainly changes public-key key establishment and signatures.

Takeaway

Find public-key use first; that is the main PQC migration target.

4

Certificates and trust chains

A certificate is a signed statement binding a public key to an identity such as a domain, organization, service, device, or signing authority. A public key alone is only data; the certificate explains who the key represents.

A trust chain usually starts at a trusted root, passes through one or more intermediates, and ends at a leaf certificate used by a website, API, device, or signing workflow. Verifiers check signatures, names, validity periods, usage rules, revocation information, and policy.

PQC changes certificates because new algorithms can mean new identifiers, larger keys, larger signatures, larger certificate chains, and legacy parser failures. PKI migration is therefore an ecosystem problem.

RootHigh-trust anchor.
IntermediateOperational signer.
LeafWebsite, API, device, or code key.
VerifierBrowser, OS, device, or service.

Concrete example

A browser may trust a root CA, which signs an intermediate CA, which signs a website certificate. If an old client cannot parse the new PQC signature identifier, the connection can fail even if the server is configured correctly.

Common pitfall

Do not migrate only a leaf certificate while ignoring roots, intermediates, clients, CT logs, proxies, and automation.

Takeaway

PQC PKI readiness means the whole trust chain can issue, carry, parse, and verify the new material.

Banking website

Challenge: Identify which cryptographic jobs happen before a customer sees account data.

Recommended answer: Certificate validation authenticates the bank; key establishment creates a session secret; symmetric encryption protects traffic; integrity checks detect tampering.

Firmware update

Challenge: Decide which job matters most when the update file is public.

Recommended answer: Signature verification and integrity matter most because the device must know the file is authentic and unchanged.

Medical archive

Challenge: Decide whether harvest-now-decrypt-later matters.

Recommended answer: Yes. Long confidentiality lifetime makes future decryption risk relevant even if attackers cannot decrypt today.
