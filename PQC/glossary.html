Glossary

Plain-English definitions of every key term in the course. Search below or scan the cards.

Authentication

Proving who a party is, or that it holds a particular secret. The security job most affected by quantum computing because it often relies on public-key crypto.

Confidentiality

Keeping content unreadable to outsiders. Provided mainly by symmetric encryption (e.g. AES), which quantum computers weaken only modestly.

Integrity

The ability to detect any change to data. Provided by hashes and MACs.

Non-repudiation

A property of signatures: later proof that a specific private key signed something, so the signer cannot credibly deny it.

Symmetric cryptography

Encryption using secret key material shared by authorized endpoints (for example AES-GCM). Shor does not structurally break it; generic quantum search changes margins, so parameters follow the required security category and policy.

Public-key (asymmetric) cryptography

Uses a public key and a private key. Enables internet-scale trust: identifying strangers, agreeing on secrets in the open, and signatures. The primary target of PQC migration.

Qubit

The basic unit of quantum information. Unlike a bit, it can be in a superposition of 0 and 1 until measured.

Superposition

A qubit's ability to be a controlled blend of 0 and 1 at once, described by amplitudes, until measurement forces a single outcome.

Logical qubit

An error-corrected qubit built from many noisy physical qubits so it behaves reliably enough for long computations.

Shor's algorithm

A quantum algorithm that efficiently solves factoring and discrete logarithms, breaking RSA, Diffie-Hellman, and elliptic-curve cryptography.

Grover's algorithm

A quantum search algorithm with a roughly square-root speedup. Halves the effective strength of symmetric keys/hashes, so longer keys restore safety.

CRQC

Cryptographically Relevant Quantum Computer: a fault-tolerant machine large and reliable enough to break deployed public-key cryptography in practical time. Does not yet exist.

Q-day

The hypothetical day a CRQC becomes available to attackers. Its date is uncertain. Use dated scenarios: X+Y>Z tests migration runway, while max(0, X−Z) estimates the confidentiality exposure window for data captured today.

Harvest now, decrypt later

An attack that copies encrypted data today and decrypts it after a CRQC exists. Makes long-lived secrets urgent even before Q-day.

KEM (Key Encapsulation Mechanism)

A scheme where a sender uses a recipient's encapsulation key to produce a ciphertext and shared secret material; the recipient decapsulates it. A protocol-defined KDF derives operational keys from that material. ML-KEM is the NIST-standardized PQC KEM.

ML-KEM (FIPS 203)

Module-Lattice KEM, derived from CRYSTALS-Kyber. The NIST standard for post-quantum key establishment.

ML-DSA (FIPS 204)

Module-Lattice Digital Signature Algorithm, derived from CRYSTALS-Dilithium. The general-purpose PQC signature standard.

SLH-DSA (FIPS 205)

Stateless Hash-based Digital Signature Algorithm, derived from SPHINCS+. A conservative signature standard resting only on hash security; large signatures.

HQC

A code-based KEM selected by NIST in 2025 as a backup to ML-KEM, providing algorithm diversity on a different mathematical assumption.

FN-DSA / Falcon

A lattice (NTRU) signature scheme known for compact signatures, being finalized as FN-DSA. Attractive where signature size matters, but tricky to implement safely.

Hybrid

Running a classical and a post-quantum mechanism together so the result is secure if either one holds. The common, lower-risk transition mode.

Crypto-agility

The ability to change algorithms, parameters, identifiers, and providers without re-architecting the system. The capability that makes future changes feasible.

Certificate

A signed statement binding a public key to an identity (domain, org, device), vouched for by a trusted authority.

Trust chain / PKI

A chain from a protected root CA through intermediates to a leaf certificate, verified end to end. PQC affects the whole chain, not just the leaf.

Constant-time

Code whose execution time and memory access do not depend on secret values, preventing timing side-channel leaks. Essential for KEM decapsulation and signing.

CBOM

Cryptographic Bill of Materials: an inventory of cryptographic assets and uses (algorithms, keys, certificates, protocols, libraries, hardware) and their relationships.

HSM / KMS / TPM / Secure element

Hardware or managed services that generate and protect keys. They must support PQC algorithms and sizes before you can use PQC keys there - often the true bottleneck.

eUICC

A secure UICC that can manage downloadable profiles. Depending on the architecture it may be embedded, integrated, or supplied in another supported form factor; capacity and compute vary by product.

LPA (Local Profile Assistant)

Device software that coordinates consumer eSIM profile download and relays RSP messages between the SM-DP+ and eUICC.

SM-DP+

The Subscription Manager Data Preparation+ server that prepares, protects, binds, and delivers consumer eSIM profile packages through the RSP architecture.

SUCI / SUPI

In 5G, the SUPI is the permanent subscriber identifier; SUCI is its concealed form, protected today with elliptic-curve public-key crypto - a PQC candidate.

Dual-signing

Attaching both a classical and a PQC signature to an artifact during transition, so old and new verifiers both work until the classical one can be dropped.
