1. Why cryptography exists

Cryptography is used when systems cannot assume the network, storage layer, or remote party is trustworthy. The goal is to preserve confidentiality, detect tampering, prove identity, and create accountability.

Concept flow
ConfidentialityProtect the content from readers who do not have the right key.
IntegrityDetect unauthorized changes to data or messages.
AuthenticationProve the identity of a server, user, device, or service.
Non-repudiationUse digital signatures to prove that a private-key holder approved something.

2. Keys and algorithms

An algorithm is the public recipe. A key is the secret or private value that makes the recipe secure. Strong systems assume attackers know the algorithm and still cannot break it without the key.

Relationship map
Symmetric key

Same key protects and opens data. Fast and used for bulk encryption.

Public/private key pair

Public key can be shared. Private key must be protected.

Session key

Temporary symmetric key negotiated for one connection or session.

Key lifecycle

Generation, storage, rotation, backup, revocation, and destruction.

3. Hashes, MACs, and signatures

Hashes create fixed-size fingerprints. MACs prove integrity when both sides share a secret. Signatures prove authenticity using public/private key pairs.

Decision matrix
Hash

One-way fingerprint used in integrity checks and signatures.

MAC

Message authentication code using a shared secret.

Signature

Private key signs; public key verifies.

Certificate

A signed identity statement that binds a public key to a name.

4. TLS and PKI in plain English

When a browser connects to a secure website, certificates authenticate the site, key exchange establishes a shared secret, and symmetric encryption protects the session. PQC affects the public-key and certificate parts most directly.

Browser asks

What is your identity and public key?

Certificate chain

A CA signs the site certificate, and the browser trusts the CA.

Key establishment

Both sides create a shared session secret.

Encrypted traffic

Bulk data uses symmetric encryption for speed.
